## Notes for next agent

- `pgvector` extension MUST be present before `drizzle-kit migrate` runs the first schema migration
  — the generated `0000_*.sql` already includes `CREATE EXTENSION IF NOT EXISTS vector`.
- `DrizzleDbClientImpl` defaults `prepare: false` to stay compatible with pgbouncer transaction-mode
  (Railway's default). Flip to `true` only when connecting direct-to-Postgres or via session-mode
  pooler.
- `DEFAULT_EMBEDDING_DIMENSIONS = 1536` matches OpenAI `text-embedding-3-small`. If the platform
  adopts a different model, bump this AND generate a fresh migration — pgvector columns are
  dimension-locked.
- `InMemoryEmbeddingStore` does cosine similarity in JS; perfectly fine for unit tests with <100
  rows, inappropriate for anything larger — do NOT reach for it in production.
- DI tokens hoisted 2026-04-24: `DB_DEPENDENCY_KEY`, `USER_REPOSITORY_DEPENDENCY_KEY`, and
  `EMBEDDING_STORE_DEPENDENCY_KEY` now alias
  `dependencyKeys.global.{DB,USER_REPOSITORY,EMBEDDING_STORE}`.
- `apps/api` routers still throw `NOT_IMPLEMENTED` pending domain-model wiring; see
  `docs/prd-status/apps/api.md` for next contiguous unit of work.
- Update this file and `docs/prd-status/matrix.md` on every change to `packages/db/**` or
  `apps/*/db` wiring.
