# @t/db migrations

Drizzle Kit-generated SQL migrations for Railway Postgres + `pgvector`.

## Files

- `0000_*.sql` — generated by `drizzle-kit generate` from `src/entities/schemas/*.ts`. The first
  migration includes `CREATE EXTENSION IF NOT EXISTS vector` so it is safe to run on a fresh
  Postgres.
- `meta/` — journal + snapshots managed by Drizzle Kit. Do not edit by hand.

## Running

```bash
# Spin up local Postgres (pgvector image)
docker compose -f docker-compose.db.yml up -d

# Apply migrations
export DATABASE_URL="postgresql://postgres:postgres@localhost:5433/template_test"
bun run --cwd packages/db db:migrate
```

For Railway, set `DATABASE_URL` in the service env and run the same command from a one-off job / CI
step. Railway Postgres ships `pgvector` out of the box.

## Regenerating from schema

After editing `src/entities/schemas/*.ts`:

```bash
bun run --cwd packages/db db:generate
```

Drizzle Kit writes a numbered `.sql` file alongside a snapshot in `meta/`. Review the diff before
committing.
